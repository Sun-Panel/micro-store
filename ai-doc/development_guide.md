## 📋 项目简介

**项目名称**：微应用商店（Sun-Panel Micro App Store）
**项目定位**：Sun-Panel 平台的微应用（插件）管理、分发、审核平台
**微应用类型**：基于 Lit + WebComponents 的轻量级组件

**核心价值**：
- 为用户提供丰富的微应用生态
- 为开发者提供便捷的发布渠道
- 为平台提供可扩展的插件系统

## 📚 文档使用流程

### 开始对话前
```
1. 让我阅读 ai-doc/development_guide.md（此文件）
2. 告诉我要开发的功能
```

### 开发过程中
```
参考 ai-doc/development_plan.md 跟踪进度
- 查看当前任务状态
- 了解下一步计划
- 确认优先级
```

### 需要查看详情时
```
参考其他文档：
- ai-doc/app_store.md - 查看数据表设计、API 定义
- ai-doc/project_overview.md - 查看业务流程、角色权限
- ai-doc/project_structure.md - 查看目录结构、开发流程
```

### 完成一个功能后
```
在 ai-doc/development_plan.md 标记完成状态：
- 将 [ ] 改为 [x]
- 添加完成日期
- 更新"当前状态"章节
```

### 开发过程中遇到新内容
```
根据内容类型判断是否需要更新文档：

需要更新文档的情况：
✅ 发现新的数据表字段或表 → 更新 app_store.md 数据表章节
✅ 新增 API 接口 → 更新 app_store.md API 列表章节
✅ 新增业务流程 → 更新 project_overview.md 业务流程章节
✅ 新增角色或权限变更 → 更新 project_overview.md 角色体系章节
✅ 项目结构有变化 → 更新 project_structure.md 目录结构章节
✅ 发现新的开发任务 → 更新 development_plan.md 添加任务项
✅ 技术方案有调整 → 在对应文档中补充说明

不需要更新文档的情况：
❌ 代码细节调整（变量名、注释等）
❌ 临时性的调试代码
❌ 性能优化（除非影响架构）
❌ UI 样式微调

更新原则：
- 保持文档简洁，避免过度详细
- 重点关注接口定义、数据结构、业务流程
- 记录重要的技术决策和设计思路
- 及时更新，避免遗忘
```

## 📝 开发规范速查

### 后端开发步骤
```
1. 检查模型是否存在（models/）
2. 创建 API 控制器（api/api_v1/）
3. 创建路由配置（router/）
4. 测试接口
```

### 前端开发步骤
```
1. 定义类型（typings/）
2. 创建 API 接口（api/）
3. 创建页面组件（views/）
4. 注册路由（router/）
5. 添加多语言（locales/）
```

### 代码风格
- **后端**：Go 标准风格，方法第一个参数为 `*gorm.DB`
- **前端**：Vue 3 Composition API + TypeScript
- **命名**：驼峰命名，遵循现有代码风格

## 💡 开发建议

### 一次性完整开发一个模块
建议按模块顺序开发：
```
分类管理（API+前端）
  ↓
开发者管理（API+前端）
  ↓
微应用管理（API+前端）
  ↓
版本管理（API+前端）
  ↓
用户端功能（API+前端）
  ↓
审核功能（API+前端）
```

### 先 API 后前端
先完成后端 API，测试通过后再开发前端页面。

### 增量式开发
每完成一个功能就测试，确保功能正常后再开发下一个。
