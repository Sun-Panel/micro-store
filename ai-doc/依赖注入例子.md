## 原来的方式（在 `biz/config.go`）

```go
// biz/config.go
package biz

import (
    "strings"
    "sun-panel/global"
)

type ConfigType struct{}

func (c *ConfigType) GetCustomStylePath() string {
    // ❌ 直接依赖 global
    customPath := global.Config.GetValueString("base", "custom_style_path")
    if customPath == "" {
        customPath = strings.TrimSuffix(c.GetWebPath(), "/") + "/custom"
    }
    return customPath
}

func (c *ConfigType) GetWebPath() string {
    return global.Config.GetValueStringOrDefault("base", "web_path")
}

// 使用方式：在 biz 层直接调用
func SomeBusinessFunc() {
    path := biz.Config.GetCustomStylePath()
    fmt.Println(path)
}
```

## 新的方式（依赖注入）

### 1. 创建 `lib/config/path.go`

```go
// lib/config/path.go
package config

import (
    "strings"
    "sun-panel/lib/iniConfig"
)

// PathProvider 路径提供者，依赖 IniConfig 接口
type PathProvider struct {
    config *iniConfig.IniConfig
}

// NewPathProvider 构造函数，注入配置对象
func NewPathProvider(cfg *iniConfig.IniConfig) *PathProvider {
    return &PathProvider{config: cfg}
}

func (p *PathProvider) GetCustomStylePath() string {
    // ✅ 只依赖 iniConfig，不依赖 global
    customPath := p.config.GetValueString("base", "custom_style_path")
    if customPath == "" {
        customPath = strings.TrimSuffix(p.GetWebPath(), "/") + "/custom"
    }
    return customPath
}

func (p *PathProvider) GetWebPath() string {
    return p.config.GetValueStringOrDefault("base", "web_path")
}
```

### 2. 在 `biz` 层使用

```go
// biz/A_ENTER.go
package biz

import (
    "sun-panel/global"
    "sun-panel/lib/config"
)

var PathProvider *config.PathProvider

func InitBIZ() {
    // ✅ 在高层创建，注入 global.Config
    PathProvider = config.NewPathProvider(global.Config)
}

// biz 某个业务方法中使用
func SomeBusinessFunc() {
    // ✅ 通过注入的 PathProvider 获取路径
    path := PathProvider.GetCustomStylePath()
    fmt.Println(path)
}
```

## 关键区别对比

| 方面 | 原来的方式 | 新的方式（依赖注入） |
|------|-----------|-------------------|
| **依赖关系** | `lib/config` → `global` ❌ | `lib/config` 只依赖 `lib/iniConfig` ✅ |
| **创建位置** | `biz/config.go` | `lib/config/path.go` |
| **初始化** | 通过 `A_ENTER.go` 的 `var Config` | 通过 `InitBIZ()` 注入 |
| **依赖注入** | 无 | 有构造函数 `NewPathProvider(cfg)` |

## 架构清晰图

```
原来：
┌─────────────┐
│   global    │
│   .Config   │
└──────┬──────┘
       │
       ▼
┌─────────────────┐
│  biz/config    │ ❌ lib 不应该依赖 global
└─────────────────┘

现在：
┌─────────────┐
│   global    │
│   .Config   │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  biz 层     │ 创建时注入
│  InitBIZ()  │─────────┐
└─────────────┘         │
                        ▼
               ┌─────────────────┐
               │ lib/config     │ ✅ 只依赖 lib/iniConfig
               │ .PathProvider  │
               └────────┬────────┘
                        │
                        ▼
               ┌─────────────────┐
               │ lib/iniConfig  │
               │  .IniConfig    │
               └─────────────────┘
```

这样 `lib/config` 就不依赖 `global` 了！🎉
