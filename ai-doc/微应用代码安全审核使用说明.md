# 微应用代码安全审核使用说明

## 概述

微应用代码安全审核功能支持两种模式：
1. **三方平台审核**：使用 sun-api 服务端进行专业代码安全审核
2. **本地静态分析**：使用内置的静态代码分析作为备用方案

## 配置说明

在配置文件中添加以下配置以启用三方平台审核：

```ini
[security_audit]
# 三方平台 API 地址（sun-api 服务端地址）
platform_url = https://your-sun-api-host.com

# API 密钥（用于身份验证）
api_key = your_api_key_here

# API 密钥（可选，如果需要）
api_secret = your_api_secret_here
```

## API 接口规范

### 请求接口

**端点**: `POST /api/v1/security/audit`

**请求头**:
```
Authorization: Bearer {api_key}
Content-Type: application/json
```

**请求体**:
```json
{
  "files": {
    "src/main.js": "console.log('hello world');",
    "src/utils.ts": "export const helper = () => {};",
    "package.json": "{\"name\": \"my-app\"}"
  },
  "scan_options": {
    "check_level": "standard",
    "language": "javascript,typescript"
  }
}
```

**请求参数说明**:
- `files`: 对象，键为文件路径（相对路径），值为文件内容
- `scan_options`: 扫描选项
  - `check_level`: 检查级别（standard/deep）
  - `language`: 扫描语言（javascript,typescript,vue,html,css）

### 响应格式

**成功响应**:
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "isPassed": true,
    "scanId": "scan_20240108_123456",
    "score": 85,
    "highRiskCount": 0,
    "mediumRiskCount": 1,
    "lowRiskCount": 2,
    "reportUrl": "https://platform.com/reports/scan_20240108_123456",
    "vulnerabilities": [
      {
        "id": "VULN_001",
        "title": "潜在的 XSS 漏洞",
        "description": "使用了 innerHTML，可能导致跨站脚本攻击",
        "severity": "medium",
        "location": "src/main.js",
        "lineNumber": 15,
        "remediation": "使用 textContent 或创建 DOM 节点来替代 innerHTML"
      }
    ],
    "scanTime": "2026-04-08T11:30:00Z",
    "error": ""
  }
}
```

**响应字段说明**:
- `code`: 响应码（0=成功，非0=失败）
- `msg`: 响应消息
- `data`: 审核结果数据
  - `isPassed`: 是否通过审核
  - `scanId`: 扫描任务ID
  - `score`: 安全评分（0-100）
  - `highRiskCount`: 高风险漏洞数量
  - `mediumRiskCount`: 中风险漏洞数量
  - `lowRiskCount`: 低风险漏洞数量
  - `reportUrl`: 审核报告URL
  - `vulnerabilities`: 漏洞详情列表
  - `scanTime`: 扫描时间
  - `error`: 错误信息（如果有）

**错误响应**:
```json
{
  "code": 40001,
  "msg": "API key 无效",
  "data": null
}
```

## 使用示例

### Go 代码示例

```go
package main

import (
    "fmt"
    "sun-panel/biz"
)

func main() {
    // 创建审核器实例
    auditor := &biz.microAppAudit{}

    // 同步调用（不推荐）
    result, err := auditor.CodeSecurityAudit("/path/to/microapp")
    if err != nil {
        fmt.Printf("审核失败: %v\n", err)
        return
    }

    if result.IsPassed {
        fmt.Printf("审核通过！评分: %d\n", result.Score)
    } else {
        fmt.Printf("审核未通过。评分: %d\n", result.Score)
        fmt.Printf("高风险: %d, 中风险: %d, 低风险: %d\n",
            result.HighRiskCount, result.MediumRiskCount, result.LowRiskCount)
    }
}
```

### 异步调用（推荐）

```go
package main

import (
    "fmt"
    "sun-panel/biz"
)

func main() {
    // 创建审核器实例
    auditor := &biz.microAppAudit{}

    // 异步调用审核
    go func(microAppPath string) {
        result, err := auditor.CodeSecurityAudit(microAppPath)
        if err != nil {
            fmt.Printf("审核失败: %v\n", err)
            return
        }

        // 处理审核结果
        handleAuditResult(result)
    }("/path/to/microapp")

    // 继续执行其他任务...
}

func handleAuditResult(result *biz.SecurityAuditResult) {
    if result.IsPassed {
        fmt.Printf("审核通过！扫描ID: %s\n", result.ScanID)
    } else {
        fmt.Printf("审核未通过，请修复以下问题：\n")
        for _, vuln := range result.Vulnerabilities {
            fmt.Printf("  - [%s] %s\n", vuln.Severity, vuln.Title)
            fmt.Printf("    位置: %s:%d\n", vuln.Location, vuln.LineNumber)
            fmt.Printf("    建议: %s\n", vuln.Remediation)
        }
    }
}
```

## 本地静态分析功能

当没有配置三方平台或 API 调用失败时，系统会自动使用内置的本地静态分析功能。

### 检测项

1. **硬编码密钥和密码**（高风险）
   - 检测模式：password, api_key, secret, token, private_key
   - 扣分：20分

2. **危险函数使用**（高风险）
   - 检测模式：eval(), Function(), setTimeout(), setInterval()
   - 扣分：15分

3. **console.log 语句**（低风险）
   - 检测模式：console.log
   - 扣分：5分

4. **XSS 风险**（中风险）
   - 检测模式：innerHTML, document.write
   - 扣分：10分

### 审核通过标准

- 安全分数 >= 60
- 高风险漏洞数量 = 0
- 中风险漏洞数量 <= 2

## API 客户端说明

本项目使用 `cnb.cool/hslr-s/go-pkg/oauth2-go/client` 包提供的 HTTP 客户端。

### 创建客户端

```go
import "cnb.cool/hslr-s/go-pkg/oauth2-go/client"

// 创建 API 客户端
apiClient := client.NewAPIClient(platformURL, 30) // 30秒超时
```

```go

// 构造请求数据

```

## 注意事项

1. **推荐异步调用**：代码安全审核可能耗时较长，建议使用 goroutine 异步调用
2. **降级机制**：当三方平台不可用时，会自动降级到本地静态分析
3. **配置验证**：使用前请确保配置文件中正确设置了平台地址和 API 密钥
4. **超时设置**：默认超时时间为 30 秒，可在配置文件中调整
5. **文件大小限制**：单个文件最大 10MB，超过限制的文件会被跳过

## 后续优化建议

1. **结果缓存**：对相同哈希值的包使用缓存的审核结果
2. **增量扫描**：仅扫描发生变更的文件
3. **自定义规则**：支持配置自定义的安全检查规则
4. **审核队列**：实现审核任务队列，支持批量审核
5. **Webhook 通知**：审核完成后通过 Webhook 通知开发者

## 故障排查

### 问题：总是使用本地分析

**可能原因**：
1. 配置文件中未设置 platform_url
2. 配置文件中未设置 api_key

**解决方法**：
检查配置文件，确保正确配置了 `security_audit` 段

### 问题：API 调用失败

**可能原因**：
1. 平台地址不正确
2. API 密钥无效
3. 网络连接问题
4. 平台服务不可用

**解决方法**：
1. 检查日志中的错误信息
2. 验证配置文件中的平台地址和 API 密钥
3. 测试网络连接
4. 联系平台管理员确认服务状态

### 问题：审核通过但仍有漏洞

**说明**：
根据审核通过标准，只要分数 >= 60、无高风险、中风险 <= 2，即可通过。低风险漏洞不影响审核通过。

**建议**：
建议开发者修复所有漏洞以提高代码安全性。
