## 一、开发时间
2026-03-31

## 二、需求分析
1. 统计微应用的下载次数和安装次数
2. 支持按统计数据排序（下载量、安装量）
3. 在应用列表中展示统计信息
4. 支持管理员和开发者查看统计数据

## 三、技术方案

### 3.1 数据库设计
在 `micro_app` 表添加了两个计数器字段：
- `download_count` - 下载次数（INT，默认 0）
- `install_count` - 安装次数（INT，默认 0）

**优点**：
- 查询性能高，无需 JOIN 查询
- 实现简单，维护成本低
- 适合中小规模应用

### 3.2 架构设计
采用 **计数器字段 + Redis 缓存** 混合方案：

```
用户下载/安装
    ↓
记录明细到数据库（micro_app_download / micro_app_install）
    ↓
Redis 计数器 +1（INCR 命令，高性能）
    ↓
定时任务同步 Redis 计数到数据库（可选，当前未实现）
```

**Redis Key 设计**：
- 下载计数：`micro_app:download:{app_id}`
- 安装计数：`micro_app:install:{app_id}`

**降级策略**：
- Redis 故障时自动降级到直接更新数据库
- 保证数据不丢失

### 3.3 分层架构

#### Biz 层（业务逻辑）

## 七、注意事项

1. **Redis 配置**：项目已集成 Redis，如果未配置会自动降级到直接更新数据库
2. **数据库迁移**：GORM 会自动创建新字段，无需手动执行 SQL
3. **缓存过期**：Redis 计数器不会过期，需要通过定时任务同步
4. **并发安全**：Redis 的 INCR 命令是原子操作，无需加锁
